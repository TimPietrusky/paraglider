Getting started

Clone this repository and use it as a starting point.

git clone git@github.com:pixelass/npm-boilerplate.git my-npm-module
cd my-npm-module
yarn

Reset git

cd my-npm-module
rm -rf .git
git init

Run package config

babel-node config

This will start a prompt in your console (similar to npm init)

The following files will be updated with your info.

  • readme.md
  • LICENSE
  • package.json

This is a one-time only config. After creation all config files are removed.

Developing

Use yarn <command>

  • start: starts the dev server and builds the required files
  • test: runs test and lints files
  • dev: starts the dev server and watches the required files
  • babel: generates lib from source
  • build: builds all files from source
  • watch: builds and watches all files from source
  • lint: lints JavaScript files
  • release: release new version using "standard-version"

Github pages

Everything inside the ./demo folder will be used as your Github page. Simply use the ./docs folder as a Github page. This option can be found on the settings page of your repository.

Views

Pages are built from the ./views folder. Every file will render a page (underscore files are excluded e.g. _mixins.pug).

JavaScript

Make sure to have a JavaScript file with the same name inside the ./demo folder.

./views/index.pug ./demo/index.js ./views/demo.pug ./demo/demo.js

Assets

All files inside ./demo/assets folder will be available in the ./docs/assets folder

While developing the same pages are served from an express app. The routes can be found inside ./scripts/routes.js and ./scripts/serve.js. The logic is simply and makes it easy to host multiple demos. If you need a different pattern you should adjust this part of the boilerplate.

What's included?

Transforms

  • Babel
    • stage 2
    • es2015
  • Browserify

You can change the rules inside the package.json file.

  • babel: {<SETTINGS>}
  • browserify: {<SETTINGS>}

defaults

{
  "babel": {
    "presets": [
      "es2015",
      "stage-2"
    ]
  },
  "browserify": {
    "transform": [
      "babelify"
    ]
  }
}

Browser matrix

  • Browserlist

You can change the rules inside the package.json file.

  • browserlist: {<SETTINGS>}

defaults

{
  "browserslist": [
    "last 2 Chrome versions",
    "last 2 ChromeAndroid versions",
    "last 2 Edge versions",
    "last 2 iOS versions",
    "last 2 Safari versions",
    "last 2 Firefox versions"
  ]
}

Coding style

  • CSS: stylelint
    • browser-features: stylelint-no-unsupported-browser-features
  • JS: xo

You can change the rules inside the package.json file.

  • xo: {<SETTINGS>}
  • stylelint: {<SETTINGS>}

defaults

{
  "xo": {
    "space": true,
    "semicolon": false
  },
  "stylelint": {
    "extends": "stylelint-config-standard",
    "plugins": [
      "stylelint-no-unsupported-browser-features"
    ],
    "rules": {
      "indentation": 2,
      "number-leading-zero": null,
      "plugin/no-unsupported-browser-features": [true]
    }
  }
}

Code quality

  • bithound

You can change the rules inside the .bithound file.

defaults

{
  "ignore": [
    "**/node_modules/**",
    "**/**.min.**",
    "**/dist/**",
    "**/docs/**",
    "/index.js"
  ],
  "test": [
    "**/spec/**"
  ],
  "critics": {
    "lint": {
      "engine": "none"
    },
    "wc": {
      "limit": 5000
    }
  }
}

Testing

  • Ava
  • Coveralls
  • tap-nyan (for some fun)

You can change the rules inside the package.json file.

  • ava: {<SETTINGS>}

defaults

{
  "ava": {
    "files": [
      "src/**/spec/*.js"
    ],
    "source": [
      "src/**/*.js"
    ],
    "presets": [
      "@ava/stage-4",
      "@ava/transform-test-files"
    ],
    "failFast": true,
    "tap": true,
    "require": [
      "babel-register"
    ],
    "babel": "inherit"
  }
}

Documentation

  • jsdoc
  • hopsdoc
  • Inch

You can change the rules inside the jsdoc.json file.

defaults

{
  "tags": {
    "allowUnknownTags": true,
    "dictionaries": ["jsdoc"]
  },
  "plugins": [
    "plugins/markdown"
  ],
  "templates": {
    "cleverLinks": true,
    "monospaceLinks": true
  },
  "source": {
    "include": ["src", "README.md"],
    "includePattern": "\\.js$",
    "excludePattern": "(node_modules/|docs|coverage|lib)"
  },
  "opts": {
    "encoding": "utf8",
    "destination": "./docs/api",
    "private": false,
    "recurse": true,
    "template": "node_modules/hopsdoc"
  }
}

Hopsdoc is a fully responsive theme that allows custom themes. You can simply remove it and load a different theme.

yarn remove hopsdoc

Inch gives a report of documentation coverage and quality.
